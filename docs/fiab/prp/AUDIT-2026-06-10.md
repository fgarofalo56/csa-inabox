# CSA Loom — Reconciled Gap Audit (2026-06-10)

This audit reconciles five ledgers — operator asks (atlas+memory), specs (12 experience PRPs + Palantir migration + v1/v2 PRPs + Master Scorecard), delivered (400 git commits / 400 merged PRs), live surfaces (editor/pane/route inventory), and UI quality (18 graded surfaces) — into a single grounded gap analysis.

**Verdict rule:** an ask counts as DELIVERED only if the delivered + live-surface ledgers show a REAL backend implementation (`useQuery`/`fetch` → `/api/*` → real Azure/Cosmos/TDS/ARM). Spec-only, UI-only, hard-coded-data, and disabled-with-tooltip surfaces are BACKLOG.

## Executive Summary

The overwhelming majority of CSA Loom's specified surface is genuinely shipped: ~400 merged PRs deliver real-backend implementations across all 12 experiences, with 75 registered item-type editors that are overwhelmingly wired to real ADLS/Synapse/ADX/AAS/Cosmos/ARM backends. The Azure-native-default mandate (no-fabric-dependency) is broadly honored, and the admin/governance/marketplace/RTI/data-engineering experiences are largely production-grade. The biggest hard gaps are a small cohort of **workspace-level panes that still render hard-coded module constants with no backend** — `monitor-hub.tsx`, `onelake-catalog.tsx`, `activator.tsx`, `semantic-model.tsx`, `real-time-hub.tsx`, and `deployment-pipelines.tsx` (pane) — at least two of which (`warehouse/page.tsx` → WarehousePane, and the onelake/activator routes) are reachable in production and are direct no-vaporware violations. A second-tier gap is a set of honest-gate "deferred to v3.x" sub-features inside otherwise-real editors (Cosmos script authoring, APIM subscription state/key-regen, geo geometry inspector + ADF param wiring, VectorStore similarity test, Ontology entity-binding, Plan approval-workflow writeback, paginated-report embed, Avro compatibility check). Spec items that remain genuinely unbuilt cluster in Databricks Unity Catalog write-path + DLT, deep AI Foundry (fine-tuning/evals/observability), several Master-Scorecard service admin blades, the Palantir L/XL migration features (Workshop/Slate/OSDK/Apollo), and most of the Wave-4 docs/tutorials/marketing/workshop deliverables. UI quality is generally B/B- with strong token discipline in admin/governance panes, but cross-cutting debt (raw `<table>` in 10 files, ~2,100 inline `style={}`, hard-coded px, clipping `maxHeight`, dead tabs) is widespread.
