# materialized-lake-view — parity with Microsoft Fabric Materialized Lake Views

**Source UI:** Fabric lakehouse → Materialized lake views tab + lineage "Manage" view + Schedule pane.
- https://learn.microsoft.com/fabric/data-engineering/materialized-lake-views/overview-materialized-lake-view
- https://learn.microsoft.com/fabric/data-engineering/materialized-lake-views/create-materialized-lake-view (Spark SQL reference)
- https://learn.microsoft.com/fabric/data-engineering/materialized-lake-views/create-materialized-lake-view-pyspark (PySpark `@fmlv`)
- https://learn.microsoft.com/fabric/data-engineering/materialized-lake-views/data-quality (CHECK … ON MISMATCH FAIL|DROP)
- https://learn.microsoft.com/fabric/data-engineering/materialized-lake-views/tutorial (medallion + lineage + schedule)

CSA Loom builds this **100% Azure-native** — there is no dependency on a real
Microsoft Fabric capacity or workspace (`.claude/rules/no-fabric-dependency.md`).
The MLV materializes a managed **Delta table on the DLZ ADLS Gen2 lake**, refreshes
via a **Synapse Spark batch** (and an **ADF "Refresh materialized lake view"
pipeline** for scheduling), and tracks **cross-workspace lineage in Cosmos**.

Zero ❌. The only ⚠️ is drop/rename which is handled by Loom's standard item
lifecycle controls (the generic item route), not a bespoke MLV control.

## Azure-native backend summary (no Fabric)

- **Storage:** ADLS Gen2 Delta under `materialized-lake-views/<view>/Tables/<schema>/<view>/`
  in the chosen medallion container (`LOOM_{BRONZE,SILVER,GOLD,LANDING}_URL`).
- **Compute:** Synapse Spark batch (Livy) running the generated PySpark driver
  (`LOOM_SYNAPSE_WORKSPACE` + `LOOM_SYNAPSE_SPARK_POOL`). Full refresh only —
  matching the PySpark MLV limitation documented by Microsoft.
- **Refresh orchestration:** ADF "Refresh materialized lake view" pipeline
  (`LOOM_SUBSCRIPTION_ID` / `LOOM_DLZ_RG` / `LOOM_ADF_NAME`); a Web activity
  calls back into `/refresh` (MSI-auth) — `LOOM_CONSOLE_BASE_URL` for the
  public callback host.
- **Lineage:** Cosmos `thread-edges` container (PK `/tenantId`) — Loom's own
  store, no OneLake / Fabric lineage tenant.
- **Query:** Synapse Serverless `OPENROWSET FORMAT='DELTA'`.

Every backend is reachable with `LOOM_DEFAULT_FABRIC_WORKSPACE` UNSET. Each
missing Azure prerequisite surfaces as an honest Fluent MessageBar naming the
exact env var / role.
