# Lock Files (`requirements/*.lock`)

CSA-0032 — **reproducible builds**. Each `*.lock` file is a fully-resolved,
hash-pinned snapshot of one `pyproject.toml` optional-dependency extra (plus
the base install). They are generated by [`pip-compile`][pip-compile] directly
from `pyproject.toml` via `--extra=<name>` (pip-tools >=7 treats
`pyproject.toml` as a first-class input — no intermediate `*.in` files are
required). The locks are the **source of truth** for:

- Docker image builds (portal backend/frontend, Azure Functions)
- CI dependency installs (`test.yml`, `frontend-test.yml`, etc.)
- SBOM generation (`sbom.yml`)
- Trivy filesystem CVE scans (`trivy.yml`)

## Files

| Lock file | Extra | Purpose |
|-----------|-------|---------|
| `base.lock` | *(none)* | Base install (bare `csa-inabox`; currently empty runtime deps) |
| `dev.lock` | `dev` | ruff, pytest, mypy, bandit, pre-commit, type stubs |
| `governance.lock` | `governance` | PyYAML, structlog, Azure Monitor ingestion |
| `functions.lock` | `functions` | Azure Functions SDK + Azure service SDKs |
| `platform.lock` | `platform` | AI integration, Databricks, Synapse, RAG loaders |
| `portal.lock` | `portal` | FastAPI, Uvicorn, Pydantic, JWT, OpenTelemetry |
| `bff.lock` | `bff` | MSAL, itsdangerous, redis (BFF auth mode) |
| `postgres.lock` | `postgres` | SQLAlchemy, asyncpg, psycopg, alembic |
| `copilot.lock` | `copilot` | pydantic-ai, OpenAI, AI Search, MCP, SSE |
| `streaming.lock` | `streaming` | Event Hubs, Schema Registry, Event Grid, Cosmos |

## Regenerating locks

```bash
# Regenerate every lock file (preferred — one command)
./scripts/update-locks.sh

# Or a single extra
./scripts/update-locks.sh portal

# Or invoke pip-compile manually (must be run from repo root)
pip-compile \
    --generate-hashes \
    --resolver=backtracking \
    --strip-extras \
    --allow-unsafe \
    --no-emit-index-url \
    --extra portal \
    --output-file requirements/portal.lock \
    pyproject.toml
```

See [`docs/SUPPLY_CHAIN.md`](../docs/SUPPLY_CHAIN.md) for the full workflow,
SBOM publication details, and CVE incident-response procedure.

## Relationship to `pyproject.toml`

The top-level declaration stays in `pyproject.toml` (PEP 621) so that
`pip install -e .[portal]` continues to work for contributors. The lock
files in this directory are **derived mechanically** from those extras via
`pip-compile --extra=<name> pyproject.toml`. This keeps `pyproject.toml`
canonical and eliminates drift between the two.

## Do not hand-edit `*.lock`

Edit `pyproject.toml`, then run `./scripts/update-locks.sh`. The lock
files are fully mechanical outputs — any manual change will be overwritten
on the next regeneration and will cause the SBOM/Trivy jobs to drift.

[pip-compile]: https://pip-tools.readthedocs.io/en/stable/
