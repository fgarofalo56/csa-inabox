# Functional Audit — Data Integration (pipelines, dataflows, copy, mirroring, linked services, IRs, datasets)

**Auditor pass:** 2026-06-26 · **Area:** `data-integration` · **Method:** UI control → BFF route → backend Azure call trace per surface + no-vaporware greps. Code-read only (no live deploy in this worktree).

## Headline

Data-integration is the **strongest area audited so far** — no `F` (vaporware) and no `D` (stubbed) surfaces. Every editor's primary action traces to a real Azure backend (ADF ARM REST, Synapse, Azure SQL TDS, ADLS Gen2, ADF-CDC). The no-vaporware greps came back clean (no `return []`/`MOCK_`/`useState(MOCK)`/dead `onClick` in the data-integration editors). The defects that exist are (a) a **Next.js 15 async-`params` regression cluster** on four `data-pipeline` sub-routes, (b) one **honest-but-real parity gap** (Mapping Data Flow data preview/debug is not wired), and (c) a **duplicate lifecycle control** UX wart on the mirrored-database editor.

## No-vaporware grep results (data-integration scope)

- `return []` / `return {}` on a non-error path: **none** in the data-integration editors or their primary routes. The two hits (`deployment-pipelines/.../pipeline-store.ts:166`, `mounted-adf/[id]/route.ts:47`) are honest `catch`-fallbacks for a 404 / per-section failure.
- `MOCK_` / `SAMPLE_` / `useState(MOCK…)`: **none**.
- `TODO` / `FIXME` / `Coming soon` / disabled-with-tooltip: **none** in the editors (the two `data-pipeline-editor.tsx` "SAMPLE" hits are comments about `LOOM_SAMPLE_ADLS` env, not stub data).
- Dead `onClick` / `onClick={() => {}}`: **none**.

## Net assessment

Data integration is **production-grade across the board** on the primary actions. To reach full A/A+ parity:

1. **P1 — fix the async-`params` cluster** (`data-pipeline` debug/evaluate/output/triggers). Mechanical 4-route change; removes runtime warnings now and prevents a hard break on Next 16. Debug is a user-facing ribbon button, so this is the highest-leverage fix.
2. **P1 — build Mapping Data Flow data preview/debug** (`createDataFlowDebugSession`/`executeDataFlowDebugCommand`). This is the one genuine feature-parity gap vs the real ADF/Fabric data-flow designer; currently an honest gate.
3. **P2 — de-duplicate the mirrored-database start/stop controls** and **UAT-confirm the mirrored-databricks action routes** to promote both to A.

No surface should be removed from the catalog — none are vaporware.
